# ViceLab Mini Brand Guide (B6)
> Version: 1.0 | Locked: 2026-04-24 | Owner: Brand Agent

---

## 1. Design Principles

- **Dark-first**: All surfaces built on `#0A0A0A`. Never white or light backgrounds.
- **Gradient identity**: Each product owns its gradient pair. Gradients go on text, wordmarks, and key UI elements — never on backgrounds.
- **Motion minimal**: Subtle transitions only. No bouncing, no heavy animations.
- **Mobile-first readability**: Every layout decision starts at 375px and scales up.
- **Clean, not cold**: Dark ≠ bleak. Use gradient accents to bring warmth and product identity.

See `colours.md` for full hex values, CSS variables, and Tailwind classes.

### Product Gradient Map

| Product | Identity | Gradient |
|---------|----------|---------|
| ViceLab | Electric Blue | `#1D4ED8 → #3B82F6` |
| SIV | Amber / Signal Gold | `#F59E0B → #F97316` |
| ASA | Violet / Deep Knowledge | `#7C3AED → #8B5CF6` |
| Matrix | Crimson / Risk Red | `#DC2626 → #EF4444` |
| VibeGuard | Blue-Green Teal | `#00C6A3 → #0066FF` |
| Cooked Pilot | Magenta / Neon Pink | `#FF0099 → #CC00FF` |

### Colour Usage Rules

1. **Gradient text** — apply `bg-gradient-to-r` + `bg-clip-text text-transparent` for wordmarks and hero headlines
2. **Solid on dark** — use product primary for icon fills, card borders, subtle bg tints
3. **No gradient backgrounds** — never fill a full section or card bg with a gradient
4. **System colours** — use shared system colours for all non-product UI (body text, borders, states)
5. **One product per section** — never mix two product gradients in the same visible block

## 3. Typography Scale

All type uses the system sans-serif stack: `Inter, -apple-system, sans-serif`.

| Level | Use Case | Size | Weight | Line Height | Tailwind |
|-------|----------|------|--------|-------------|----------|
| Display | Hero H1 | 56–72px | 700 | 1.05 | `text-6xl lg:text-7xl font-bold` |
| Heading 1 | Page title | 40–48px | 700 | 1.1 | `text-4xl lg:text-5xl font-bold` |
| Heading 2 | Section title | 28–36px | 600 | 1.15 | `text-3xl lg:text-4xl font-semibold` |
| Heading 3 | Card title | 20–24px | 600 | 1.2 | `text-xl lg:text-2xl font-semibold` |
| Body Large | Lead paragraph | 18px | 400 | 1.6 | `text-lg leading-relaxed` |
| Body | Standard text | 16px | 400 | 1.6 | `text-base leading-relaxed` |
| Small | Labels, captions | 14px | 400 | 1.5 | `text-sm` |
| Eyebrow | Section labels | 12px | 600 | 1.4 | `text-xs font-semibold uppercase tracking-widest` |

### Typography Rules

- **Gradient headlines only** — apply gradient to H1/H2 on product pages, not body copy
- **No centre-aligning body copy** — left-aligned body text only on mobile
- **Eyebrows** — always `uppercase tracking-widest text-xs` in product primary colour or muted grey
- **Short paragraphs** — max 3 sentences per paragraph, then break
- **No bold inside paragraphs** — use a new heading or bullet instead

## 4. Spacing System

### Base Unit

`4px` base unit. All spacing is a multiple of 4.

### Layout Grid

| Breakpoint | Container | Columns | Gutter |
|------------|-----------|---------|--------|
| Mobile (< 640px) | 100% | 1 | 16px |
| Tablet (640–1024px) | 100% − 48px | 2 | 24px |
| Desktop (> 1024px) | max 1200px | 3–4 | 32px |

```css
.section-container {
  max-width: 1200px;
  margin: 0 auto;
  padding: 0 24px;
}
@media (min-width: 1024px) {
  .section-container { padding: 0 48px; }
}
```

### Section Spacing

| Level | Value | Tailwind | Use |
|-------|-------|----------|-----|
| Page section | 96–128px | `py-24 lg:py-32` | Between major page sections |
| Sub-section | 64px | `py-16` | Between related content blocks |
| Component | 32–48px | `gap-8 lg:gap-12` | Between cards, list items |
| Element | 16–24px | `gap-4 lg:gap-6` | Inside components |
| Tight | 8px | `gap-2` | Icon + label, badge + text |

### Component Spacing

**Cards:**
```
padding: 24px (p-6)
border-radius: 12px (rounded-xl)
gap between cards: 24px (gap-6)
```

**Buttons:**
```
padding: 12px 24px (py-3 px-6) — standard
padding: 10px 20px (py-2.5 px-5) — compact
border-radius: 8px (rounded-lg)
```

**Sections:**
```
heading → body gap: 16px (mb-4)
body → CTA gap: 32px (mt-8)
section headline → eyebrow gap: 8px (mb-2)
```

## 5. Logo / Text Identity Rules

Applies when no SVG logo exists (SIV, ASA, Matrix):

- Render wordmark as gradient text using product colour pair
- Font: `font-bold` or `font-semibold`, size matches context (nav = `text-xl`, hero = `text-3xl`+)
- No icon or box around text — clean, flat, gradient only
- Never duplicate text label next to an SVG logo (applies to VibeGuard, Cooked Pilot, ViceLab)

SVG logos (vicelab.svg, vibeguard.svg, cookedpilot.svg):
- Root path: `/logos/filename.svg`
- Header height: ~36px
- Left-aligned in nav
- No resizing inconsistency — equal visual weight across product logos
- Never mix two logos in one section

## 6. Component Patterns

```jsx
<section className="py-24 lg:py-32">
  <div className="max-w-[1200px] mx-auto px-6 lg:px-12">
    <p className="text-xs font-semibold uppercase tracking-widest text-[var(--product-primary)] mb-2">
      Eyebrow Label
    </p>

    </h1>

      One to two sentence value proposition. Short. Clear.
    </p>
    <a href="/product" className="inline-block py-3 px-6 rounded-lg bg-gradient-to-r from-[#START] to-[#END] text-white font-semibold">
      Primary CTA
    </a>
  </div>

```

```jsx
<div className="rounded-xl border border-[#222222] bg-[var(--product-surface)] p-6 hover:border-[var(--product-primary)] transition-colors">
  <p className="text-xs font-semibold uppercase tracking-widest text-[var(--product-primary)] mb-3">
    Product Label
  </p>
  <h3 className="text-xl font-semibold text-white mb-2">Card Title</h3>
  <p className="text-sm text-[#A3A3A3] leading-relaxed">
    Supporting description. Keep to 2 lines max.
  </p>
</div>
```

## 8. Tone of Voice (per product)

| Product | Tone | Voice Notes |
|---------|------|-------------|
| ViceLab | Authoritative, clear | Platform-level, no jargon |
| SIV | Clinical, precise | Data-forward, intelligence framing |
| ASA | Academic, analytical | Knowledge-depth, not cold |
| Matrix | Direct, urgent | Risk-aware, no-nonsense |
| VibeGuard | Calm, trustworthy | Safety-first, reassuring |
| Cooked Pilot | Bold, irreverent | Community voice, not corporate |

Rules:
- Short sentences. Active voice.
- No filler: remove "simply", "just", "easy", "powerful"
- Never use exclamation marks in product copy (Cooked Pilot social excepted)
- Lead with the outcome, not the feature
