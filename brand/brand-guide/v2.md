## 1. Design Language

All products share a single design language:
- **Dark-first**: `#0A0A0A` page background always
- **Gradient identity**: each product owns one gradient; it is the primary brand expression
- **Text as intentional design**: until SVG logos exist, styled text is the logo ŌĆö not a placeholder
- **No gradients on backgrounds**: gradients on strokes, text, icons, CTAs only
- **Cyberpunk-adjacent**: neon on black, tight geometry, precision spacing

### Product Gradient Map

| Product | Identity | Gradient CSS |
|---------|----------|-------------|
| **SIV** | Amber / Signal Gold | `linear-gradient(90deg, #F59E0B 0%, #F97316 100%)` |
| **ASA** | Violet / Deep Knowledge | `linear-gradient(90deg, #7C3AED 0%, #8B5CF6 100%)` |
| **Matrix** | Crimson / Risk Red | `linear-gradient(90deg, #DC2626 0%, #EF4444 100%)` |
| **VibeGuard** | Blue-Green / Compliance Teal | `linear-gradient(90deg, #00C6A3 0%, #0066FF 100%)` |
| **Cooked Pilot** | Magenta / Neon Pink | `linear-gradient(90deg, #FF0099 0%, #CC00FF 100%)` |

### Shared System Colours

| Role | HEX | Usage |
|------|-----|-------|
| Page Background | `#0A0A0A` | All products, all pages |
| Text Primary | `#F9FAFB` | Body text on dark |
| Text Muted | `#6B7280` | Captions, labels, secondary info |
| Border | `#1F2937` | Card outlines, dividers |
| Status OK | `#22C55E` | Success states |
| Status Warn | `#F59E0B` | Warning states |
| Status Critical | `#EF4444` | Errors, critical risk |

### Colour Usage Rules

1. **Gradient = identity marker** ŌĆö use on the element that names the product (wordmark text, logo stroke, section eyebrow, CTA button)
2. **Primary = solid fills** ŌĆö buttons, badges, inline highlights when gradient is too noisy
3. **Surface = card background** ŌĆö each product has a slight tint toward its identity hue
4. **Never** apply a product gradient to a section background
5. **Never** mix two product gradients on the same element
6. **Text on primary**: use `#0A0A0A` for SIV (amber) and VibeGuard (teal); `#FFFFFF` for all others

## 3. Type Scale

Next.js + Tailwind. All sizes are Tailwind utility classes.

| Level | Class | Size | Weight | Usage |
|-------|-------|------|--------|-------|
| Display | `text-6xl` | 60px | `font-extrabold` | Hero headlines only |
| H1 | `text-4xl md:text-5xl` | 36ŌĆō48px | `font-bold` | Page titles |
| H2 | `text-3xl md:text-4xl` | 30ŌĆō36px | `font-bold` | Section headings |
| H3 | `text-2xl` | 24px | `font-semibold` | Card titles, sub-sections |
| H4 | `text-xl` | 20px | `font-semibold` | Feature labels |
| Body Large | `text-lg` | 18px | `font-normal` | Lead copy, intro paragraphs |
| Body | `text-base` | 16px | `font-normal` | General body text |
| Body Small | `text-sm` | 14px | `font-normal` | Secondary info |
| Caption | `text-xs` | 12px | `font-normal` | Timestamps, labels |
| Eyebrow | `text-xs tracking-widest uppercase font-semibold` | 12px | ŌĆö | Section labels above H2 |

- **Eyebrow colour**: product gradient via `bg-clip-text text-transparent`, or primary HEX solid
- **Line height**: `leading-tight` for headings, `leading-relaxed` for body
- **Max line width**: `max-w-prose` (65ch) for body copy blocks

## 5. Logo / Text Identity Rules

**Current phase: text as intentional design element**

- SIV, ASA, Matrix ŌĆö styled text wordmarks. Not placeholders. Designed to look complete.
- VibeGuard, Cooked Pilot, ViceLab ŌĆö SVG logos at `public/logos/*.svg`

**Wordmark styling for SIV / ASA / Matrix:**
```jsx
<span className="bg-gradient-to-r from-[#START] to-[#END] bg-clip-text text-transparent font-bold tracking-tight">
  PRODUCT NAME
</span>
```

**SVG logo rules:**
- `height: 36px, width: auto` in header
- Root path: `/logos/filename.svg`
- No text duplication next to logo
- No mixing product logos in one section
- Equal visual weight across logos
